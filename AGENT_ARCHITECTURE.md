# 🤖 InvoHydra Fully Automated SEO Pipeline Architecture

This document defines the 5-Agent Architecture for the InvoHydra SEO Pipeline. It is designed to be a fully autonomous "set-and-forget" inbound marketing machine.

Currently, **Agents 3 and 4 are implemented**. To achieve 100% automation, **Agents 1, 2, and 5** need to be built.

---

## 🕵️‍♂️ Agent 1: The Keyword Discoverer (To Be Built)
**Function**: Automatically finds new, relevant long-tail keywords based on a broad seed topic. 
**Why it's needed**: It replaces the manual process of coming up with keywords and removes the need for the static `manual_keywords.json` file.

### Implementation Steps:
1. Create a new script: `agents/discoverer.py`.
2. **Input**: A broad "Seed Topic" (e.g., "SaaS billing software in India").
3. **API Integration**: Use the **Serper API** (Google Search API) to search for this seed topic.
4. **Data Extraction**: Extract the "People Also Ask" (PAA) questions and "Related Searches" from the Serper JSON response. These make the best long-tail keywords.
5. **LLM Filtering**: Pass these raw phrases to the Groq API to quickly filter out irrelevant junk or competitor brand names.
6. **Output**: A clean list of 10-20 long-tail keywords, passed directly to Agent 2.

---

## 📊 Agent 2: The Difficulty Analyst (To Be Built)
**Function**: Analyzes the SEO competition for the discovered keywords and discards the ones that are too difficult for a new/growing site to rank for.
**Why it's needed**: Prevents the pipeline from wasting time and API credits writing articles that will be buried on Page 5 of Google because the competition (like Forbes or HubSpot) is too strong.

### Implementation Steps:
1. Create a new script: `agents/analyst.py`.
2. **Input**: The list of keywords generated by Agent 1.
3. **SERP Analysis**: For each keyword, use the **Serper API** to fetch the top 10 organic search results.
4. **Domain Checking**: Extract the domain names of the top 10 results.
5. **Filtering**: Create a simple "blacklist" of high-authority domains (e.g., `forbes.com`, `hubspot.com`, `cleartax.in`). If a keyword's top 10 results contain too many of these giant domains, discard the keyword.
6. **Output**: Save the surviving "low-hanging fruit" keywords to a JSON file (or pass them dynamically to Agent 3).

---

## 🧠 Agent 3: The Semantic Intent Clusterer (Completed)
**Function**: Takes the filtered keywords, validates them against product capabilities (using a Feature Truth Map), and clusters them based on Demand, Product Fit, and Search Intent.
**Status**: Done (`agents/planner.py`).

**How it works**: 
Uses the Groq API to enforce strict compliance boundaries. It ensures the AI never claims features the product does not support, acting as a guardrail against hallucinations. It outputs the organized clusters as structured JSON.

---

## ✍️ Agent 4: The Blog Writer (Completed - Minor Tweaks Needed)
**Function**: Conducts live competitor research and writes comprehensive, deep-dive SEO blog posts.
**Status**: Done (`agents/writer.py`).

**How it works**: 
Uses Serper to find the #1 ranking competitor for a cluster, scrapes their content using Firecrawl to identify content structure, and uses Groq API to write a highly detailed Markdown post that outperforms them.

### Suggested Tweaks for the Team:
- **Add SEO Metadata**: Update the Groq prompt to output a JSON object containing `meta_title`, `meta_description`, `url_slug`, and `markdown_body`.
- **Adjust Tone**: Increase the Groq API `temperature` slightly (e.g., from `0.3` to `0.5`) so the writing sounds more natural and less robotic.
- **Section-by-Section Generation**: To guarantee 1500+ word counts, have the LLM generate an outline first, then loop through and write each section individually.

---

## 🚀 Agent 5: The Auto-Publisher (To Be Built)
**Function**: Takes the final Markdown files generated by Agent 4 and automatically publishes them to your website CMS (WordPress, Webflow, Ghost, etc.).
**Why it's needed**: Removes the final manual bottleneck of copying, pasting, and formatting the articles into your website's backend.

### Implementation Steps:
1. Create a new script: `agents/publisher.py`.
2. **Input**: Read the Markdown files (and SEO metadata) generated by Agent 4.
3. **Format Conversion**: Convert the Markdown text into HTML (using the Python `markdown` library).
4. **API Integration**: Use your specific CMS's API (e.g., WordPress REST API or Webflow CMS API) to send a `POST` request containing the HTML, Title, and Meta Description.
5. **Publishing**: Upload the content as a "Draft" (so a human can review it before it goes live) or directly as "Published".
6. **Cleanup**: Move the successfully published `.md` files to an `archive/` folder so the pipeline doesn't accidentally publish them twice.

---

## 🔄 Final Step: The Automation Loop
Once all 5 Agents are built and chained together sequentially in `main.py`:
1. Host the script on a cloud server or GitHub.
2. Set up a **GitHub Action** or a Cron Job.
3. Schedule it to run automatically (e.g., every Monday at 3 AM). 

**The Result**: Every Monday, your pipeline wakes up, finds new un-tapped keywords (Agent 1 & 2), plans the clusters (Agent 3), writes the content (Agent 4), and uploads it to your site (Agent 5)—completely autonomously!
